# Task Checklists

## Create task

- Assign `TASK-<id>`
- Set `Status` and `Priority`
- Link Product/Architecture/Plan references
- Define in-scope and non-goals
- Write measurable acceptance criteria
- Add test plan

## Move to review

- Acceptance criteria are checked
- Changes exist in code/docs
- Verification steps executed
- Risks and mitigations updated

## Close task

- Status set to `done`
- Result explains delivered output
- Evidence attached (tests/logs/manual verification)
- Backlog entry updated
